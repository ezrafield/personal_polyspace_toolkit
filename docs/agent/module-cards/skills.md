# Module: Skills

## Responsibility
Holds the canonical C-only Polyspace workflows consumed by all supported clients.

## Rules
- Keep descriptions specific enough for model discovery.
- Use only the five compatible MCP tool names.
- Require explicit profiles and `.c` translation units.
- Prefer code fixes and preserve justification/test execution approvals.
- Put detailed fallback syntax and API references beside the owning skill.

## Tests
Frontmatter, references, tool-name allowlist, removed-surface scan, and policy scenario fixtures.

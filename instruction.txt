The key idea: 4-layer context architecture
Layer 1: Auto-loaded, very small
These are loaded automatically by the agent.
AGENTS.md
CLAUDE.md

src/module/CLAUDE.md

Use them only for:
- Project purpose
- Tech stack
- Critical commands
- Must-follow rules
- Where to find deeper docs
- Safety constraints
- Definition of done

Do not put long architecture explanations, full API specs, old decisions, examples, or troubleshooting history here. Claude uses CLAUDE.md as persistent project instructions, while Codex uses AGENTS.md with global/project/nested precedence. (Claude Code) (OpenAI Developers)
Layer 2: On-demand agent docs
These are not auto-loaded unless the agent needs them.
docs/agent/INDEX.md
docs/agent/CODEMAP.md
docs/agent/ARCHITECTURE.md
docs/agent/WORKFLOWS.md
docs/agent/module-cards/*.md

This is where token saving happens. Instead of asking Claude/Codex to explore the whole repo, you tell it:
Before editing, read docs/agent/INDEX.md.
Then read only the module-card related to the task.
Do not scan unrelated directories unless needed.

This follows the production AI system principle that the model should not be the whole system; reliable systems need context building, retrieval/tools, validation, logs, metrics, and continuous improvement around the model.

Minimal AGENTS.md
# AGENTS.md

## Project purpose
This repository is [one-sentence product/system purpose].

## Default workflow
1. Understand the task.
2. Read `docs/agent/INDEX.md`.
3. Read only the relevant module card under `docs/agent/module-cards/`.
4. Make the smallest safe change.
5. Add or update tests.
6. Run targeted tests before broad tests.
7. Summarize changed files, commands run, and remaining risks.

## Context rules
- Do not scan the whole repository unless the task requires it.
- Prefer `rg`, targeted file reads, and module cards.
- Do not load large docs unless referenced by `docs/agent/INDEX.md`.
- Keep changes scoped to the user request.

## Commands
- Install: `make install`
- Run app: `make dev`
- Unit tests: `make test-unit`
- Integration tests: `make test-integration`
- Lint: `make lint`
- Typecheck: `make typecheck`

## Code rules
- Follow existing patterns before introducing new abstractions.
- Do not add new dependencies without explicit justification.
- Do not change public APIs without updating docs/specs.
- Do not modify generated files manually.

## Definition of done
- Relevant tests pass.
- Lint/typecheck pass when applicable.
- Docs updated if behavior changes.
- Final response includes files changed, tests run, and risks.

Best practice for module cards
Each module card should be short.
# Module: Auth

## Responsibility
Handles login, refresh token, logout, and current-user lookup.

## Key files
- `src/auth/routes.py`: HTTP endpoints
- `src/auth/service.py`: auth business logic
- `src/auth/security.py`: hashing/JWT utilities
- `tests/integration/test_auth.py`: integration tests

## Public interfaces
- `AuthService.login(...)`
- `AuthService.refresh(...)`
- `get_current_user(...)`

## Rules
- Routes should not contain business logic.
- Services should not import FastAPI request objects.
- JWT secrets come from config only.

## Common tasks
- Add auth endpoint в†’ inspect `routes.py`, `service.py`, tests
- Change token behavior в†’ inspect `security.py`, `service.py`

## Known pitfalls
- Do not change refresh token expiry without updating tests.
- Do not log raw tokens.

Best workflow for Claude Code / Codex
For every task, use this protocol:
1. Ask agent to restate the goal.
2. Ask agent to read docs/agent/INDEX.md.
3. Ask agent to identify relevant module cards.
4. Ask agent to produce a short plan.
5. Let it inspect targeted files only.
6. Let it patch.
7. Run targeted tests.
8. Run broader tests only if needed.
9. Update task log.
10. Final answer: summary, files changed, tests, risks.

For bigger tasks, use this prompt:
Before editing code:
- Read AGENTS.md / CLAUDE.md.
- Read docs/agent/INDEX.md.
- Select the minimal relevant docs and files.
- Create a task note under .agent/tasks/.
- Explain your plan.
- Do not scan unrelated directories.
- After implementation, update the task note with files changed, commands run, and remaining risks.

Final recommendation
Use this architecture:
Thin instruction files
+ Agent context index
+ Module cards
+ Generated code map
+ Task logs
+ ADR/spec docs
+ Test/eval layer
+ Local traces ignored by git

This is the best balance for Claude Code and Codex today because it gives agents enough structure to act like senior engineers, but avoids the trap of auto-loading too much context. It also fits a product-level AI system mindset: context building, workflow control, validation, logging, and cost control are part of the system, not afterthoughts.
